# Розовый Кастинг

Шуточное браузерное миниприложение в стиле dating app parody для девичника: свайпаем анкеты, раскладываем по категориям `Нравится` / `Нет`, а потом выбираем одного победителя среди понравившихся.

## Запуск

1. Установить зависимости:

```bash
npm install
```

2. Запустить проект локально:

```bash
npm run dev
```

3. Собрать production-версию:

```bash
npm run build
```

4. При необходимости посмотреть собранную версию:

```bash
npm run preview
```

Примечание: даже без анкет приложение уже запускается. В этом случае оно покажет пустое состояние и подскажет, куда положить `.md`-файлы и фотографии.

## Где лежат анкеты

Анкеты и фотографии нужно класть в папку:

`src/content/profiles`

Приложение автоматически подхватит все `.md`-файлы и попытается найти к каждому фото с таким же именем.

## Как связываются анкета и фото

Связь идет по одинаковому имени файла.

Пример:

- `src/content/profiles/artyom.md`
- `src/content/profiles/artyom.jpg`

Поддерживаемые форматы изображений:

- `.jpg`
- `.jpeg`
- `.png`
- `.webp`
- `.avif`

## Формат анкеты

Используется `frontmatter` плюс основной markdown-текст:

```md
---
name: "Артем"
hobbies: ["спорт", "бар", "путешествия"]
occupation: "продакт"
style: "smart casual"
pets: "собаки"
worst_act: "может пропасть на 3 дня"
---

Люблю активный отдых, вкусно есть и красиво жить.
```

## Обязательные поля

- `name`
- `hobbies`
- `occupation`
- `style`
- `pets`
- `worst_act`
- основной markdown-текст после `frontmatter`

Если какое-то обязательное поле не заполнено или не найдено фото с таким же именем, приложение покажет предупреждение.

## Как добавить новую анкету

1. Создать `.md`-файл в `src/content/profiles`.
2. Положить рядом фотографию с тем же именем.
3. Перезапустить dev-сервер, если новый файл не подхватился автоматически.

## Что уже реализовано

- стартовый экран
- свайпы и кнопки `Нравится` / `Нет`
- прогресс по анкетам
- экран, если никто не понравился
- экран финального выбора среди понравившихся
- праздничный экран победителя
- вынесенные тексты интерфейса
- вынесенные цветовые токены темы

## Публичная ссылка через GitHub Pages

Проект подготовлен под `GitHub Pages`.

После публикации ссылка будет такого вида:

`https://username.github.io/repository-name/`

### Как опубликовать

1. Создай новый репозиторий на GitHub.
2. Загрузи туда содержимое этой папки.
3. Убедись, что основная ветка называется `main`.
4. Открой в GitHub:
   `Settings -> Pages`
5. В разделе `Build and deployment` выбери `GitHub Actions`.
6. После пуша в `main` автоматически запустится workflow деплоя.
7. Через пару минут GitHub Pages выдаст публичную ссылку.

### Что уже настроено

- `vite.config.ts` умеет собирать сайт с правильным `base` для GitHub Pages
- workflow деплоя лежит в `.github/workflows/deploy-pages.yml`
- при пуше в `main` сайт автоматически собирается и публикуется

### Если сайта не видно сразу

- проверь, что репозиторий не пустой
- проверь вкладку `Actions`, успешно ли прошел workflow
- проверь, что в `Settings -> Pages` выбран источник `GitHub Actions`
